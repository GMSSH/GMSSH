## 我可以贡献什么？

我们欢迎多种形式的贡献：

* **🔌 新应用/插件：** 使用我们的 SDK 构建新工具来解决特定的运维问题（例如，Redis GUI、日志分析器）。
* **🐛 Bug 修复：** 修复 `gmssh-plugin-sdk` 或 `gmssh-official-apps` 中的问题。
* **📝 文档：** 改进 SDK 文档或将其翻译成其他语言。
* **💡 功能请求：** 为 SDK 建议新的 API（例如，"我们需要一个访问本地文件系统的 API"）。

> **注意：** GMSSH 客户端的核心二进制文件是闭源的。请不要逆向工程或反编译核心逻辑。请专注于 JavaScript/Python/GO 扩展层。

## 开发新应用

**🚀 快速入门：直接开始编码**
想要立即构建应用？如果您希望跳过架构理论并立即开始您的开发者之旅，请点击以下链接：

[👉 开始您的开发者之旅](https://doc-dev.gmssh.com/)

**📖 理解 GMSSH 设计理念**
如果您希望在编写代码之前掌握底层架构，这里是对核心概念的深入探讨：

**1. 🏗️ GMSSH 开发工作流程**
首先，必须了解核心架构理念。GMSSH 远不止是一个 Web 服务器——它是一个复杂的进程编排系统。

**2. 🧠 核心引擎：ga_main**
在 GMSSH 的拓扑结构中，ga_main 是基石，作为系统的"中央神经系统"和路由枢纽。

区别于 LAMP：与 Apache/Nginx 通常直接处理业务逻辑不同，ga_main 被设计为轻量级、高性能的微内核。

**3. 🛡️ 进程隔离与稳定性**
ga_main 与业务逻辑完全解耦；其唯一目的是生命周期管理和流量调度。所有功能——从官方文件管理器到自定义 Nginx 管理器——都作为隔离的子进程运行。

沙箱可靠性：如果第三方插件崩溃（例如，由于内存泄漏），它只会终止该单个进程。ga_main 和其他服务不受影响。

自动恢复：系统主动监控进程并自动处理重启。

**4. 🌉 网关与协议转换**
ga_main 作为入口网关，聚合来自客户端界面（Electron/Web）的所有流量。

职责：处理请求验证（身份验证/授权）、应用 ID 解析和调度。

透明桥接：无缝地将外部 HTTP/WebSocket 协议转换为内部 IPC（进程间通信）。

**5. ⚡ 内核级数据传输**
虽然标准 TCP/IP 会产生开销（分段、校验和、防火墙过滤），但 GMSSH 利用 Unix 域套接字（UDS）实现卓越性能。数据直接在内核内存缓冲区中复制，完全绕过网络堆栈。

🚀 零拷贝性能：大幅减少 CPU 上下文切换。

⏱️ 低延迟：微秒级延迟（相比 TCP 的毫秒级），非常适合"实时终端"或"日志流"。

🔒 安全性：依赖文件系统权限。只有 ga_main 和应用所有者可以访问套接字，从根本上消除端口扫描风险。

**6. 🔄 双模式启动支持**
生产环境（推荐）：使用 UDS 以获得最大安全性和性能。

开发/调试：GMSSH 支持通过 HTTP 端口提供后端服务，便于在 Windows 上开发或在不原生支持 UDS 的跨节点部署场景中使用。

**7. 📜 JSON-RPC 2.0 协议**
在应用层，GMSSH 采用 JSON-RPC 2.0——一种无状态、轻量级的 RPC 协议。这使得后端与语言无关（Python、Go、Rust、Node.js 等）。

请求结构：

jsonrpc：版本

method：函数名

params：参数（对象/数组）

id：唯一标识符

响应结构：

返回执行结果或错误对象。

## Pull Request 指南

准备好提交您的代码了吗？太好了！

1.  **先同步：** 确保您的 fork 与 `main` 分支保持最新。
2.  **描述性标题：** 使用清晰的标题（例如，`feat: 在 Redis 插件中添加深色模式支持`）。
3.  **检查清单：**
    * [ ] 我的代码遵循本项目的代码风格（ESLint/Prettier）。
    * [ ] 我已对自己的代码进行了自我审查。
    * [ ] 我已在难以理解的地方对代码进行了注释。
    * [ ] 我已验证应用在 GMSSH Client v2.0+ 中运行顺畅。

## 应用中心发布

如果您希望您的应用在 **官方 GMSSH 应用中心** 上架（对所有用户可用）：

1.  向 `gmssh-marketplace` 注册表提交 Pull Request（或在我们的网站上提交表单）。
2.  **安全审查：** 我们的团队将审查您的代码，确保没有恶意操作（例如，窃取 SSH 密钥）。
3.  **批准：** 一旦获得批准，您的应用将被上架，您将获得 **"GMSSH 贡献者"** 徽章！
